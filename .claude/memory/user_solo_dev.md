The user is a solo developer on the Fashionista project (and by default, on other projects in this environment). No collaborators, no code reviewers, no team-wide style guide to defer to.

**How to apply:**
- Don't suggest team-oriented workflows: PR templates, CODEOWNERS, branch protection for reviewers, "what will your teammates think" framing.
- Committing personal/local config files (e.g. `.claude/settings.local.json`) is fine — there's no one else to share or not share with. Don't default to gitignoring "personal" files on solo repos.
- Style/convention decisions are the user's own — don't hedge with "your team might prefer X".
